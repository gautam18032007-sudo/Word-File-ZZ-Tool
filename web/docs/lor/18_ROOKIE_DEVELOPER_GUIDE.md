# 18. Rookie Developer Guide — LOR Module

This guide is written for developers who are new to the ZenZebra project. It explains every concept, tool, and workflow needed to understand and contribute to the LOR module.

---

## 1. Glossary of Terms

| Term | Definition |
|---|---|
| **LOR** | Letter of Recommendation — a formal document certifying an employee's tenure and recommending them for future opportunities. |
| **DOCX** | Microsoft Word document format. The system generates LOR letters as `.docx` files using templates. |
| **PDF** | Portable Document Format. Generated by converting the DOCX file using LibreOffice. |
| **Google Form** | An online form where employees submit their details (name, role, projects, etc.). |
| **Google Response Sheet** | A Google Spreadsheet automatically populated with Google Form submissions. |
| **Gemini API** | Google's AI language model API used to generate the recommendation body text. |
| **PizZip** | A JavaScript library for reading/writing ZIP archives. Used because `.docx` files are ZIP archives. |
| **Docxtemplater** | A library that finds `{{PLACEHOLDER}}` tags inside Word documents and replaces them with real data. |
| **LibreOffice** | A free, open-source office suite. Used in headless mode to convert `.docx` → `.pdf`. |
| **Next.js App Router** | The routing system used in Next.js 16. Each folder in `app/` represents a URL route. |
| **API Route** | A server-side endpoint. Files named `route.ts` inside `app/api/` folders handle HTTP requests. |
| **Sequence Number** | A counter that increments with each generated document (e.g., `0001`, `0002`). |


## 3. How to Set Up Your Local Environment

### Prerequisites
1. **Node.js** (v18 or later) — [Download](https://nodejs.org/)
2. **LibreOffice** (for PDF conversion) — [Download](https://www.libreoffice.org/download/)
3. **Google Gemini API Key** — [Get one here](https://aistudio.google.com/apikey)

### Setup Steps
```bash
# 1. Clone the repository
git clone <repo-url>
cd "CONTRACT TOOL"

# 2. Install dependencies
npm install

# 3. Configure environment
# Edit .env and add:
# GOOGLE_LOR_SHEET_ID=your-sheet-id
# GEMINI_API_KEY=your-api-key
# LIBREOFFICE_PATH=path-to-libreoffice

# 4. Start the development server
npm run dev

# 5. Open in browser
# http://localhost:3000/lor
```

## 4. How Data Flows Through the System

```text
┌────────────────┐
│  Google Form   │  Employee fills out the form online
└───────┬────────┘
        ↓
┌────────────────┐
│  Google Sheet  │  Responses are stored automatically
└───────┬────────┘
        ↓
┌────────────────┐
│  /api/sheets/  │  Our API reads the sheet data
│  lor           │
└───────┬────────┘
        ↓
┌────────────────┐
│  /lor page     │  HR sees the list of employees
│  (browser)     │  and selects one
└───────┬────────┘
        ↓
┌────────────────┐
│  Form fields   │  Details auto-fill from sheet data
│  auto-fill     │  HR can edit any field
└───────┬────────┘
        ↓
┌────────────────┐
│  /api/generate │  HR clicks "Generate AI Draft"
│  /lor/draft    │  → Gemini AI writes the letter body
└───────┬────────┘
        ↓
┌────────────────┐
│  AI Draft      │  HR reviews and edits the AI text
│  Editor        │
└───────┬────────┘
        ↓
┌────────────────┐
│  /api/generate │  HR clicks "Generate Document"
│  /lor          │  → DOCX + PDF are created
└───────┬────────┘
        ↓
┌────────────────┐
│  output/lors/  │  Files saved to disk
│  lor-history   │  Record saved to history
└────────────────┘
```

## 7. Troubleshooting

| Problem | Solution |
|---|---|
| "No Google Sheet URL or ID provided" | Add `GOOGLE_LOR_SHEET_ID` to your `.env` file |
| "Sheet is not publicly accessible" | Share the Google Sheet as "Anyone with the link — Viewer" |
| "AI service not configured" | Add `GEMINI_API_KEY` to your `.env` file |
| "PDF generation unavailable" | Install LibreOffice and set `LIBREOFFICE_PATH` in `.env` |
| Build fails with TypeScript errors | Run `npm run build` and fix the reported errors |
| Page shows "Coming Soon" | The LOR page hasn't been implemented yet — follow `16_IMPLEMENTATION_GUIDE.md` |

---

## 8. Golden Rules for New Developers

1. **Never touch Brand, Employee, or Certificate code.** The LOR module is fully isolated.
2. **Always run `npm run build`** after making changes to check for errors.
3. **Read the docs first.** Everything is documented. If something isn't clear, ask before guessing.
4. **Test with real data.** Use the actual Google Sheet with real employee responses.
5. **Keep it simple.** Follow the patterns already established in the Employee and Certificate modules.
